# CLAUDE.md — Skill Factory

Universal repository of AI coding agent skills. Each skill is a self-contained directory with a SKILL.md source of truth. A build pipeline generates platform-specific files for Claude Code, Gemini CLI, and Codex CLI.

## Repo Structure

```
skill-factory/
├── CLAUDE.md
├── README.md
├── CONTRIBUTING.md
├── LICENSE
├── .gitignore
├── package.json
├── tsconfig.json
├── src/                             # Build tooling (TypeScript)
│   ├── build.ts                     # Main build orchestrator
│   ├── validate.ts                  # Frontmatter validator
│   ├── catalog.ts                   # README catalog generator
│   └── platforms/                   # Platform adapters
│       ├── base.ts
│       ├── claude-code.ts
│       ├── gemini-cli.ts
│       └── codex-cli.ts
├── skills/                          # Source of truth — one dir per skill
│   └── <skill-name>/
│       ├── SKILL.md
│       └── evals/
│           └── evals.json
└── dist/                            # Generated + committed — ready to copy
    ├── claude-code/<skill-name>/SKILL.md
    ├── gemini-cli/<skill-name>/SKILL.md
    └── codex-cli/<skill-name>/AGENTS.md
```

## Source of Truth

`skills/<name>/SKILL.md` is canonical. The `dist/` directory is generated by `npm run build`.

## Contributor Workflow

1. Create or edit `skills/<name>/SKILL.md`
2. Run `npm run validate` to check frontmatter
3. Run `npm run build` to regenerate dist/ and README catalog
4. Commit both `skills/` and `dist/` changes
5. Submit PR

## What Gets Committed

**Tracked:**
- `skills/<name>/SKILL.md` — skill definition (source of truth)
- `skills/<name>/evals/evals.json` — evaluation cases
- `dist/` — generated platform-specific files
- `src/` — build tooling

**Excluded:**
- `node_modules/` — install with `npm install`
- `build/` — TypeScript compilation output
- `*-plan.md`, `*-downloads/`, `*-screenshots/` — dev workspace artifacts

## SKILL.md Requirements

- Must have YAML frontmatter with `name` (required) and `description` (required)
- Optional fields: `version`, `tags`, `platforms`, `author`
- Must be self-contained — no external file dependencies
- Body is platform-agnostic markdown instructions

## Supported Platforms

| Platform | Output Format | Adapter |
|----------|--------------|---------|
| Claude Code | SKILL.md (passthrough) | `src/platforms/claude-code.ts` |
| Gemini CLI | SKILL.md (passthrough) | `src/platforms/gemini-cli.ts` |
| Codex CLI | AGENTS.md (transformed) | `src/platforms/codex-cli.ts` |

## Adding a Platform

1. Create `src/platforms/<platform-id>.ts` implementing `PlatformAdapter`
2. Register in `src/platforms/index.ts`
3. Run `npm run build` to generate output
4. Update README install instructions for the new platform

## Code Standards

- Skills are markdown + JSON — no executable code in skills/
- Build tooling is TypeScript in src/
- Keep SKILL.md focused and actionable — instructions for AI agents
- Evals should cover primary use cases (3+ test cases per skill)
