## Source of Truth

`skills/<name>/SKILL.md` is canonical. The `dist/` directory is generated by `npm run build`.

## Contributor Workflow

1. Create or edit `skills/<name>/SKILL.md`
2. Run `npm run validate` to check frontmatter + platform compatibility
3. Run `npm run eval-check` to verify structural content
4. Run `npm test` to run unit tests
5. Run `npm run build` to regenerate dist/ and README catalog
6. Commit both `skills/` and `dist/` changes
7. Submit PR (CI runs all checks automatically)

## What Gets Committed

**Tracked:**
- `skills/<name>/SKILL.md` вЂ” skill definition (source of truth)
- `skills/<name>/evals/evals.json` вЂ” evaluation cases
- `dist/` вЂ” generated platform-specific files
- `src/` вЂ” build tooling

**Excluded:**
- `node_modules/` вЂ” install with `npm install`
- `build/` вЂ” TypeScript compilation output
- `*-plan.md`, `*-downloads/`, `*-screenshots/` вЂ” dev workspace artifacts

## SKILL.md Requirements

- Must have YAML frontmatter with `name` (required) and `description` (required)
- Optional fields: `version`, `tags`, `platforms`, `author`
- Must be self-contained вЂ” no external file dependencies
- Body is platform-agnostic markdown instructions

## Supported Platforms

| Platform | Output Format | Adapter |
|----------|--------------|---------|
| Claude Code | SKILL.md (passthrough) | `src/platforms/claude-code.ts` |
| Gemini CLI | SKILL.md (passthrough) | `src/platforms/gemini-cli.ts` |
| Codex CLI | AGENTS.md (transformed) | `src/platforms/codex-cli.ts` |

## Adding a Platform

1. Create `src/platforms/<platform-id>.ts` implementing `PlatformAdapter`
2. Register in `src/platforms/index.ts`
3. Run `npm run build` to generate output
4. Update README install instructions for the new platform

## Code Standards

- Skills are markdown + JSON вЂ” no executable code in skills/
- Build tooling is TypeScript in src/
- Keep SKILL.md focused and actionable вЂ” instructions for AI agents
- Evals should cover primary use cases (3+ test cases per skill)
- All build tooling changes must have unit tests (vitest)
- CI must pass before merging: validation, tests, eval-check, dist freshness
