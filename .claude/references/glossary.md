# Skill Factory Glossary

## Core Concepts

**Skill** — A self-contained AI agent instruction set. Lives in
`skills/<name>/SKILL.md` with YAML frontmatter and markdown body.
Independently useful when copied to a platform's install path.

**Adapter** — A `PlatformAdapter` implementation (`src/platforms/*.ts`)
that transforms a `ParsedSkill` into platform-specific `OutputFile`(s).
Three adapters exist: ClaudeCodeAdapter (passthrough), GeminiCliAdapter
(strips to name+description), CodexCliAdapter (converts to AGENTS.md).

**dist/** — Generated output directory. One subdirectory per platform,
each containing transformed skill files. Created by `npm run build`.
Committed to git for copy-ready distribution.

**Catalog** — Auto-generated markdown table in `README.md` between
`<!-- CATALOG:START -->` and `<!-- CATALOG:END -->` markers. Lists all
skills with name, description, platforms, and tags. Regenerated by
`src/catalog.ts` during build.

**Frontmatter** — YAML header block in SKILL.md (delimited by `---`).
Parsed by `gray-matter`. Defines skill metadata: name, description,
version, tags, platforms, author. See `references/schemas/skill-frontmatter.md`.

**Eval** — Declarative test case in `evals/evals.json`. Each eval has
a prompt, expected output, file list, and behavioral expectations.
Checked structurally by `src/eval-check.ts`, not executed automatically.

## Build Pipeline Terms

**validate** — `npm run validate` runs `src/validate.ts`. Checks
frontmatter required/optional fields and platform compatibility.

**eval-check** — `npm run eval-check` runs `src/eval-check.ts`. Checks
SKILL.md body for structural patterns (H1 heading, no credentials, etc.).

**build** — `npm run build` runs `src/build.ts`. Discovers skills,
runs all platform adapters, writes to dist/, injects catalog into README.

**verify-dist** — CI job that rebuilds and checks `git diff dist/` to
ensure committed dist/ matches the current skills/ source.
