# Changelog

Per-skill and build-tooling changes. Dates in ISO 8601.

## [playwright-autopilot 4.0.0] — 2026-04-21

**Breaking.** Skill refactored into a dispatcher that delegates to a new project-scoped subagent.

### Added
- `.claude/agents/domain-playwright-lead.md` — Playwright domain lead agent. Owns all MCP browser interaction, recon, incremental scripting, and layered debug. Tool whitelist includes `mcp__plugin_playwright_playwright__*` — physically prevents substitution of `requests` / BeautifulSoup / `httpx`.
- **Mentor consultation protocol.** Agent emits `NEEDS_MENTOR` when it hits ambiguity (selector collisions, missing credentials, unclear intent, layout drift, 3 failed debug attempts). Main thread auto-answers from conversation context or asks the human, then respawns the agent with `RESUME session <id>. Mentor answer: ...`.
- **Session state** at `.claude/agent-memory/domain-playwright-lead/sessions/<id>/state.json` (gitignored; agent auto-creates).
- **Deadlock counter owned by the mentor**, hashed on `(checkpoint + blocker_category)` rather than raw question text, with `deadlock_equivalent_to` self-declaration from the agent.
- **Round-trip ceiling:** max 10 mentor round-trips per session OR 3 deadlocks → `ESCALATE_USER`.
- 2 new eval cases (ids 9, 10) covering selector ambiguity and missing credentials.

### Changed
- `platforms:` narrowed to `[claude-code]`. The mentor-consultation pattern requires Claude Code's subagent dispatch; Gemini CLI and Codex CLI can't spawn project-scoped agents, so shipping them the dispatcher would be broken-on-arrival. If Gemini/Codex support is required, pin the skill at tag `v3.1.0`.
- Skill body shrunk from a full playbook to a thin dispatcher (trigger gate → pre-flight → dispatch → mentor loop → hand-off). All Hard Rules, Goal Lock mechanics, Smart Recon, Development Loop, Layered Debug, and Script Quality Standards now live in the agent body.

### Removed
- `dist/gemini-cli/playwright-autopilot/SKILL.md`
- `dist/codex-cli/playwright-autopilot/AGENTS.md`

### Migration
Users on v3.x get autopilot inline in the skill body. v4.x requires Claude Code and relies on the companion agent. No runtime breakage — scripts generated by v3.x remain valid; only the authoring flow changed.

## [build-tooling] — 2026-04-21

### Added
- `platforms:` frontmatter is now a **build filter**. Previously advisory; the build emitted every skill to every platform regardless of declared targets. Now `src/build.ts` skips emission for platforms not in a skill's `platforms:` array. Skills without the field continue to emit to all platforms (backwards-compatible default).
